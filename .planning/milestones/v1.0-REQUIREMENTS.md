# Milestone Requirements Archive: v1.0 - Kline Data Fetcher

## Functional Requirements
- [x] **FR1: Data Source Integration**: Download from `binance.vision`. (Validated: E2E fetch test)
- [x] **FR2: User-Defined Storage Path**: `--output-dir` command-line argument. (Validated: CLI tests)
- [x] **FR3: Data Storage Structure**: `<user_data_path>/<SYMBOL>/<KLINECSV_DATA_FILES>`. (Validated: E2E file system check)
- [x] **FR4: URL Discovery and Filtering**: Auto-discovery and `.zip.CHECKSUM` exclusion. (Validated: `BinanceVisionService` tests)
- [x] **FR5: Data Download**: Download identified ZIP files to `.tmp`. (Validated: `DownloadService` tests)
- [x] **FR6: Data Extraction and Conversion**: Extract to CSV and remove ZIP. (Validated: `ExtractionService` tests)

## Non-Functional Requirements
- [x] **NFR1: Reliability**: Resumes and retries with exponential backoff. (Validated: Network failure mocks)
- [x] **NFR2: Performance**: Concurrent processing with `p-limit`. (Validated: Visual test with 100+ files)
- [x] **NFR3: Usability**: Clear CLI and real-time progress. (Validated: Manual UAT)
- [x] **NFR4: Error Handling**: Human-friendly messages for 404/ENOTFOUND/Corrupted ZIP. (Validated: Manual error injection)
