# Phase 05-01 PLAN: Robust Download & Resume Logic

## Objective
Implement reliability features in `DownloadService` to handle transient network issues and resume interrupted downloads.

## Context
- Current `DownloadService` handles basic downloads but fails permanently on transient errors and doesn't support resuming partial files.
- Requirement NFR1 (Reliability) and NFR4 (Error Handling) depend on this.

## Strategy
1. **Retries**: Implement a retry mechanism with exponential backoff for `downloadFile`.
2. **Resume**: Add logic to check existing file size and use the `Range` HTTP header if supported by the server.
3. **Unit Tests**: Update existing tests and add new ones for retry/resume scenarios.

## Execution Steps
1.  **Enhance `DownloadService.downloadFile`**:
    - Add a `maxRetries` and `retryDelay` parameter (with defaults).
    - Implement a loop that catches network/HTTP errors and retries with backoff.
    - Check for existing file size before initiating the fetch.
    - Add `Range: bytes=${existingSize}-` to headers if `existingSize > 0`.
    - Handle `206 Partial Content` (append to file) vs `200 OK` (overwrite/start fresh).
    - Handle `416 Range Not Satisfiable` (file might be complete or corrupted).
2.  **Update `DownloadService` Tests**:
    - Mock a failed request that succeeds on retry.
    - Mock a partial download resume scenario.
3.  **Refactor `ExtractionService`**:
    - Ensure it handles corrupted zips gracefully (throws informative error).
    - Add a check for already extracted CSV files to avoid redundant work (idempotency base).

## Verification Strategy
- **Unit Tests**: `npm test tests/services/download.service.test.ts`
- **Manual Test**: Simulate an interrupted download (by killing the process) and verify it resumes correctly in the next run.
- **Error Handling**: Verify clear messages when the URL is invalid or the server doesn't support Range requests.

## Implementation Details
- Use `fs.stat` to check existing file size.
- Use `fs.createWriteStream(destPath, { flags: 'a' })` for appending.
- Use `AbortController` (optional) for timeouts.
