# Phase 05-02 PLAN: Concurrency & CLI Performance

## Objective
Enable parallel processing of multiple kline files to significantly improve performance (NFR2) and enhance CLI user experience (NFR3).

## Context
- Currently, downloads are sequential. For a symbol like BTCUSDT with 100+ months of data, this is inefficient.
- Users have no visual feedback on progress.

## Strategy
1. **Concurrency Control**: Use `p-limit` to process multiple files in parallel (default: 5 concurrent downloads).
2. **Progress Reporting**: Integrate `cli-progress` to show a multi-bar progress display.
3. **CLI Arguments**: Add `--concurrency` and `--force` flags to `commander`.
4. **Idempotency**: Check if the target CSV already exists and skip download/extraction unless `--force` is used.

## Execution Steps
1.  **Install Dependencies**:
    - `npm install p-limit cli-progress`
2.  **Update `cli.js` with Concurrency**:
    - Use `p-limit` to wrap the download/extract logic.
    - Default concurrency level of 5.
    - Add `--concurrency` option to `fetch` command.
3.  **Integrate `cli-progress`**:
    - Create a global progress bar manager.
    - Add individual bars for each file (or a single high-level bar if too many).
    - Handle terminal resize/cleanup gracefully.
4.  **Implement Skip Logic**:
    - Before starting a download, check if `${symbolDir}/${fileName}.csv` exists.
    - Log "Skipping (already exists)" and increment progress without downloading.
    - Add `--force` to overwrite existing files.
5.  **Refine CLI Output**:
    - Add a summary table at the end (Success, Failed, Skipped).
    - Use `chalk` (optional, check if available) for colors.

## Verification Strategy
- **Manual Test**: Run `fetch` with `--concurrency 10` for a symbol with many months and verify it's faster.
- **Idempotency Test**: Run twice; the second run should be almost instantaneous as it skips existing files.
- **UX Test**: Verify progress bars update correctly and don't flicker.

## Implementation Details
- Handle potential race conditions (though files are unique).
- Ensure error handling is robust within the parallel pool.
