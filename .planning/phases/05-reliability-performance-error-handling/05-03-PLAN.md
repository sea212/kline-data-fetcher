# Phase 05-03 PLAN: Advanced Error Handling & Final Polish

## Objective
Finalize error handling for corrupted files and improve CLI usability (NFR3, NFR4).

## Context
- Corrupted ZIP files or partially downloaded files can cause extraction failures.
- CLI currently uses simple `console.error` which might be too verbose or not informative enough.

## Strategy
1. **Corrupt ZIP Handling**: Catch `extract-zip` errors and explicitly handle corrupted archive errors.
2. **Error Message Clarity**: Map technical errors (e.g., `EACCES`, `ENOTFOUND`, `HTTP 404`) to human-friendly messages.
3. **Validation Polish**: Add schema-based validation (or just more robust checks) for symbols and intervals.
4. **Final Verification**: Run the tool through a series of "failure scenarios" to verify reliability.

## Execution Steps
1.  **Refactor `ExtractionService.extractAndCleanup`**:
    - Specifically catch extraction errors.
    - If corruption is detected, delete the corrupted ZIP and tell the user.
    - Provide an option or hint to re-run with `--force`.
2.  **Enhance CLI Error Handling**:
    - Implement a `formatErrorMessage(error)` utility.
    - Map `ENOTFOUND` to "Cannot connect to binance.vision (Network Issue)".
    - Map `EACCES` to "Permission denied for output directory".
    - Handle `HTTP 404` as "Symbol or Interval not found on binance.vision".
3.  **Input Validation**:
    - Add basic validation for symbols (all caps, regex).
    - Add validation for intervals (allowed list: 1m, 3m, 5m, 15m, 30m, 1h, 2h, 4h, 6h, 8h, 12h, 1d, 3d, 1w, 1M).
4.  **Final Documentation**:
    - Update `README.md` (if exists) or create a brief usage guide with new flags (`--concurrency`, `--force`).
    - Create `05-VERIFICATION.md` report.

## Verification Strategy
- **Simulate Errors**:
    - Provide a non-existent symbol (expect 404/Empty result message).
    - Provide an invalid output directory (expect "Permission denied" or "Invalid path").
    - Corrupt a ZIP manually and run extraction (expect "Corrupted archive detected").
- **Full Run**: Download a real symbol from scratch and verify total integrity.

## Implementation Details
- Avoid over-engineering; keep error mapping simple.
- Use `fs.unlinkSync` or async equivalent for cleanup after corruption.
